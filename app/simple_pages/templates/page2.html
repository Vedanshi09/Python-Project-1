Container

A docker container is a package that includes everything to run an application. It can be stopped, started, moved or even delete using Docker API or CLI. We can connect a docker container with one or more networks.

Images

A docker image is like a template that provides a list of instructions to create a docker container in order to run on Docker platform.

Project repository

Project repository in Dockers provides sharing of images in container with other member of Docker community.

Docker Command

Following are some of the Docker commands that I have used.

docker start - Starts one or more stopped containers docker stop - Stops one or more running containers docker run- Runs a command in a new container docker build – Builds an image form a Docker file docker pull – Pulls an image or a repository from a registry docker push – Pushes an image or a repository to a registry docker commit – Creates a new image from a container’s changes

Building project in Docker

Docker command can be run through Terminal without using PyCharm. For example, if we want to make a new container, we can type in the following commands in the terminal.

docker create -- name=new_container -it ubuntu docker start new_container docker exec new_container /path/to/my/command docker exec new_container /path/to/another/command
